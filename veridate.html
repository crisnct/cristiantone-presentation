VeriDate

Location-based and AI-powered dating app for real, verified connections—no fakes, no ghosting. Smart matching finds your perfect match, while AI-enhanced photos reflect their true essence. Authentic dating, made effortless.

The Problem

Many users of dating apps are complaining about not finding the right people, with whom they can have a meaningful connection.

The Goal

Our dating app will help single individuals seeking a relationship to find the right people with whom they can have a meaningful and genuine connection. We will measure effectiveness by the number of first-time in-person meetings between two matches.

UX Design Process

Empathize

  • Conduct surveys and interviews with potential users to understand their needs and pain points.
    • Survey Interview1 Interview2 Interview3 Interview4
  • Create empathy maps to visualize user emotions and experiences.
  • Create user personas based on research findings to guide app development.
  • Determine the pain points.
  • Create user journey maps to visualize the user experience and identify key touchpoints.

Define User Problems

  • Problem Statements: Articulate specific user pain points into clear, actionable problems.
  • Hypothesis Statements: Formulate testable assumptions about potential solutions to user problems.
  • Value Propositions: Define the core benefits and unique selling points of the app.

Ideate

  • Competitive Audit: create competitor analysis to understand the strengths and weaknesses of existing dating apps.
    • Competitive Audit Research Competitive Audit Report
  • Create information architecture to define the app's structure and navigation.
  • Create paper wireframes to visualize the app's layout and functionality.

Prototype

  • Create low-fidelity prototype using Figma.
  • Plan a usability study with the same users to gather feedback on the prototype.
  • Create a survey
  • Create affinity diagram to organize user feedback and insights.
  • Update low-fidelity prototype
  • Create a new survey
  • Create affinity diagram to organize user feedback and insights.
  • Create high-fidelity prototype using Figma.
  • Plan a usability study with the same users to gather feedback on the prototype.
  • Create a survey
  • Create affinity diagram to organize user feedback and insights.
  • Update high-fidelity prototype
  • Compare the first and second versions of the high-fidelity prototype to highlight improvements and changes based on user feedback.

Accessibility

Considerations for inclusive design, ensuring the app is usable by individuals with diverse abilities (e.g., highly readable typography, strong color contrast, large buttons, centered elements for left/right-handed users).
